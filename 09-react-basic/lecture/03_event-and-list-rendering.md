# 9주차 - React 기초 3: 이벤트와 리스트 렌더링

## 이번 주 목표

- React에서 이벤트를 처리하는 방법을 이해할 수 있다.
- 배열 데이터를 화면 목록으로 출력할 수 있다.
- `key`가 왜 필요한지 설명할 수 있다.

## 왜 이 실습을 하나요?

TODO 앱은 입력, 클릭, 목록 출력이 계속 일어나는 구조입니다.
따라서 React 이벤트 처리와 리스트 렌더링은 최종 프로젝트 전에 반드시 익혀야 하는 핵심 개념입니다.

## 준비물

- `09-react-basic/example/react-counter/`
- `10-todo-react/example/todo-react-complete/`

## 이번 주 파일 설명

- `09-react-basic/example/react-counter/`: 버튼 클릭 이벤트 예제입니다.
- `10-todo-react/example/todo-react-complete/`: 배열 데이터를 목록으로 출력하는 구조를 보여줍니다.

## 코드에서 꼭 볼 부분

- `onClick`: 클릭 이벤트를 연결합니다.
- 배열 `map()`: 여러 항목을 반복 출력합니다.
- `key`: 각 항목을 구분하는 값입니다.

## 실습 순서

1. counter 예제의 버튼 이벤트를 읽습니다.
2. 배열 데이터를 화면에 반복 출력하는 예시를 떠올려봅니다.
3. TODO 앱에서 왜 `key`가 필요한지 생각해봅니다.

## 학생 실습 미션

1. 배열 데이터를 목록으로 출력하는 구조를 말로 설명해보세요.
2. `key`가 없으면 왜 불편한지 생각해보세요.
3. 버튼 이벤트와 목록 렌더링이 TODO 앱에서 어떻게 쓰일지 적어보세요.

## 체크리스트

- [ ] React 이벤트 처리 방식을 이해했다.
- [ ] 배열 데이터를 반복 출력하는 개념을 이해했다.
- [ ] `key`의 목적을 설명할 수 있다.

## 자주 하는 실수

- HTML 속성처럼 이벤트를 생각하는 경우
- 배열을 바로 화면에 넣기만 하고 구조화하지 않는 경우
- `key`를 그냥 아무 값이나 넣는 경우
