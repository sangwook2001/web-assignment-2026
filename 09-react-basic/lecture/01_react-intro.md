# 9주차 - React 기초 1: React가 왜 필요한가

## 이번 주 목표

- React가 등장한 이유를 설명할 수 있다.
- HTML, CSS, JavaScript만으로 만들 때의 불편을 이해할 수 있다.
- 컴포넌트 기반 개발의 장점을 알 수 있다.

## 왜 이 실습을 하나요?

화면이 작을 때는 순수 JavaScript로도 충분할 수 있습니다.
하지만 페이지가 커지고 기능이 늘어나면 코드가 복잡해지고 재사용이 어려워집니다.
React는 이런 문제를 줄이기 위해 널리 사용되는 도구입니다.

## 준비물

- `09-react-basic/example/react-counter/` 예제
- `09-react-basic/example/react-profile-card/` 예제

## 이번 주 파일 설명

- `09-react-basic/example/react-counter/`: state 변화를 보는 예제입니다.
- `09-react-basic/example/react-profile-card/`: 컴포넌트와 props를 보는 예제입니다.

## 코드에서 꼭 볼 부분

- React는 UI를 작은 컴포넌트로 나누어 관리합니다.
- 상태가 바뀌면 화면도 함께 갱신됩니다.
- 같은 구조를 반복해서 쓸 때 컴포넌트 재사용이 편리합니다.

## 실습 순서

1. `09-react-basic/example/react-counter/`를 먼저 열어봅니다.
2. 버튼을 누를 때 숫자가 바뀌는 흐름을 읽습니다.
3. `09-react-basic/example/react-profile-card/`로 넘어가 컴포넌트 분리를 확인합니다.
4. 두 예제가 어떤 점에서 순수 HTML 방식과 다른지 생각해봅니다.

## 학생 실습 미션

1. React가 필요한 이유를 한 문단으로 적어보세요.
2. 반복되는 UI를 컴포넌트로 나누면 어떤 장점이 있는지 적어보세요.
3. counter 예제에서 화면이 다시 그려지는 순간을 설명해보세요.

## 체크리스트

- [ ] React의 등장 이유를 설명할 수 있다.
- [ ] 컴포넌트 분리의 장점을 이해했다.
- [ ] 상태 변화와 화면 변화의 관계를 이해했다.

## 자주 하는 실수

- React를 단순히 "새 문법" 정도로만 이해하는 경우
- HTML을 완전히 대체하는 것으로 오해하는 경우
- 화면 구조와 상태 관리를 따로 생각하지 않는 경우

## 한 줄 정리

React는 복잡한 화면을 더 작게 나누고 더 안정적으로 관리하기 위한 도구입니다.
