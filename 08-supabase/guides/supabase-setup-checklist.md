# Supabase 설정 체크리스트

실습 시작 전에 아래 항목을 직접 확인합니다.

## 프로젝트

- [ ] `New project`로 Supabase 프로젝트를 만들었다.
- [ ] 프로젝트 대시보드에 정상적으로 들어갈 수 있다.

## 스키마

- [ ] `todo` 테이블을 만들었다.
- [ ] `id`를 기본 키로 만들었다.
- [ ] `content` 컬럼을 만들었다.
- [ ] `is_done` 컬럼을 만들고 기본값을 `false`로 설정했다.
- [ ] `created_at` 컬럼을 만들고 기본값을 `now()`로 설정했다.

## API 값

- [ ] `Settings > API`에 들어갈 수 있다.
- [ ] `Project URL` 위치를 확인했다.
- [ ] `anon public key` 위치를 확인했다.
- [ ] `service_role` 키를 프론트엔드 코드에 넣으면 안 된다는 점을 이해했다.

## 코드 일치 여부

- [ ] 코드의 테이블 이름과 Supabase 테이블 이름이 같다.
- [ ] 코드의 컬럼 이름과 Supabase 컬럼 이름이 같다.

## 최종 확인

- [ ] Supabase에서 데이터를 조회할 수 있다.
- [ ] 데이터를 추가했을 때 테이블에 실제 행이 생성된다.
