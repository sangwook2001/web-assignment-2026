# 8주차 - Supabase 1: 서비스 개요

## 이번 주 목표

- Supabase가 어떤 역할을 하는 서비스인지 설명할 수 있다.
- 프론트엔드와 데이터 저장소의 관계를 이해할 수 있다.
- 왜 TODO 앱에 데이터베이스가 필요한지 설명할 수 있다.

## 왜 이 실습을 하나요?

지금까지는 화면 중심으로 작업했습니다.
하지만 실제 서비스는 데이터를 저장하고, 다시 불러오고, 수정할 수 있어야 합니다.
Supabase는 초보자도 비교적 쉽게 데이터베이스 기능을 경험할 수 있게 도와줍니다.

## 준비물

- Supabase 계정
- 브라우저
- `08-supabase/example/guestbook-app/` 또는 `08-supabase/starter/supabase-connection-practice/` 폴더

## 이번 주 파일 설명

- `08-supabase/example/guestbook-app/`: Supabase 연결 완성 예제입니다.
- `08-supabase/starter/supabase-connection-practice/`: 학생이 직접 연결을 완성하는 starter입니다.

## 코드에서 꼭 볼 부분

- Supabase는 데이터베이스와 연결된 API 역할을 함께 제공합니다.
- 프론트엔드 코드는 Supabase에 직접 요청을 보내 데이터를 읽고 쓸 수 있습니다.

## 실습 순서

1. Supabase가 왜 필요한지 먼저 이해합니다.
2. TODO 앱에서 어떤 데이터를 저장해야 하는지 생각합니다.
3. 예제 폴더의 화면 구조를 확인합니다.
4. 나중에 어떤 부분이 Supabase와 연결될지 짐작해봅니다.

## 학생 실습 미션

1. TODO 앱에서 저장해야 할 데이터 3가지를 적어보세요.
2. 브라우저 화면과 데이터 저장소의 역할 차이를 정리해보세요.
3. Supabase가 없는 경우 어떤 불편이 생길지 설명해보세요.

## 체크리스트

- [ ] Supabase의 큰 역할을 이해했다.
- [ ] TODO 앱에 데이터 저장이 왜 필요한지 설명할 수 있다.
- [ ] 프론트엔드와 데이터 저장의 차이를 구분할 수 있다.

## 자주 하는 실수

- Supabase를 단순한 코드 라이브러리 정도로만 이해하는 경우
- 화면에 보이는 것과 저장되는 데이터를 같은 것으로 생각하는 경우
- 데이터베이스 없이도 새로고침 후 데이터가 남는다고 오해하는 경우

## 한 줄 정리

Supabase는 웹앱이 데이터를 저장하고 다시 불러오게 해주는 백엔드 도구입니다.
