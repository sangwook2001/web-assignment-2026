# 8주차 - Supabase 2: DB와 인증 기초

## 이번 주 목표

- 데이터베이스의 기본 구조를 이해할 수 있다.
- 테이블, 컬럼, 행의 차이를 설명할 수 있다.
- TODO 테이블 구조를 읽을 수 있다.

## 왜 이 실습을 하나요?

데이터베이스를 모르면 Supabase 코드를 읽을 때도 무엇을 저장하는지 감이 잘 오지 않습니다.
이번 단계에서는 먼저 "어떤 데이터를 어떤 칸에 넣는가"를 이해하는 것이 중요합니다.

## 준비물

- Supabase 프로젝트
- TODO 테이블 설계 메모

## 이번 주 파일 설명

- `/guides/supabase-guide.md`: 테이블 구조와 설정 방향을 정리한 문서입니다.
- `/guides/supabase-setup-checklist.md`: 설정 전 점검 항목입니다.

## 코드에서 꼭 볼 부분

- 테이블 이름: `todo`
- 컬럼: `id`, `content`, `is_done`, `created_at`
- `id`: 각 항목을 구분하는 값
- `content`: 할 일 내용
- `is_done`: 완료 여부
- `created_at`: 생성 시각

## 실습 순서

1. TODO 앱에서 필요한 데이터 항목을 적어봅니다.
2. 각 항목이 컬럼으로 어떻게 들어가는지 정리합니다.
3. `todo` 테이블 구조를 읽어봅니다.
4. 어떤 컬럼이 꼭 필요한지 다시 생각해봅니다.

## 학생 실습 미션

1. `id`가 왜 필요한지 설명해보세요.
2. `is_done`에 어떤 값이 들어갈지 예를 들어보세요.
3. `created_at`이 없는 경우 어떤 불편이 생길지 적어보세요.

## 체크리스트

- [ ] 테이블과 컬럼의 개념에 대해 설명할 수 있다.
- [ ] TODO 테이블의 4개 컬럼 역할을 이해했다.
- [ ] 한 줄 데이터와 전체 테이블을 구분할 수 있다.

## 자주 하는 실수

- 테이블 이름과 컬럼 이름을 혼동하는 경우
- 완료 여부를 텍스트만으로 저장하려는 경우
- `id` 없이 목록을 관리하려는 경우

## 한 줄 정리

데이터베이스는 표처럼 생각하면 이해가 쉽고, 컬럼 설계가 앱 구조를 결정합니다.
